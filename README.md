# Qwen2.5-7B LoRA SFT + DPO

LoRA experiments on Qwen2.5-7B-Instruct using LLaMA-Factory (RTX 3090 24GB).

## M1 — Identity change

identity(91) + alpaca_zh_demo(1000), 3 epochs, eval_loss=1.3785.
Demo: [results/before_after_demo.md](results/before_after_demo.md)

## M2 — Self-bootstrap e-commerce CS dataset

300 dialogues generated by Qwen2.5-7B itself (10 seeds + few-shot, 0 API). LoRA rank=8, target=all, 5 epochs, eval_loss=0.9958. 5/5 unseen queries learn CS style ("您好"/简洁/订单号请求/共情).
Demo: [results/m2/before_after_demo.md](results/m2/before_after_demo.md)

## M3 — DPO preference alignment

DPO on dpo_zh_demo (300 preference triples), LoRA rank=8, β=0.1, lr=5e-6, 3 epochs.

- eval_loss = 0.6759 (< ln(2)=0.693)
- rewards/accuracies = 0.5667 (>0.5, direction correct)
- rewards/margins = 0.0412 (chosen−rejected gap)

**Insight**: DPO on already-RLHF'd Instruct model = marginal polishing, NOT dramatic style change (use SFT for the latter). Output becomes more compact and slightly better aligned.

Demo: [results/m3/before_after_demo.md](results/m3/before_after_demo.md)
